### 意識したこと・わからなかったこと
- `head`内にリセットCSSを追加
　→今後`style.css`と連携する際、写す媒体によってデザインが変わらないため。（ちょっと気が早かった）
- お問い合わせボタンに`button`を使用
　→用途がわからずお問い合わせセクションにリンク。ボタンの作り方がわからず、見たことのあった`button`を使用。
　　→`a`に変更。ページ遷移は`a`、送信などの操作は`button`という認識。JavaScript。
- 各セクションを`section`で分けた
　→`div`の乱用ではなく、アクセシビリティやSEOの観点からセマンティックなコードを書くことを意識。
- 各セクションへのリンクに`id`を使用
　→やり方がわからず調べた。
　→参考デザインではグローバルナビゲーションの「お客様の声」はリンク先が「スタッフ紹介」だったため、リンク先に合わせて変更。
　　→`id="1"`から`id=product"`などに変更。実務では数字ではなく意味のある言葉でセクションを管理すると知った。
- メインビジュアルとキャッチフレーズ/コピーを`div`で囲った
　→まとまりをつくろうと考えたが、意味なく`div`を使用した。
　　→`section`に変更。キャッチコピーがセクションだという認識がなかった。
- Googleマップの埋め込みに`iframe`を使用
　→やり方がわからず調べた。サイズ調整はCSSで行えることを知り、参考デザインのマップURLのみコピペした。
- お問い合わせセクションの連絡先に`mailto:`と`tel:`を使用
　→やり方ががわからず調べた。
- お問い合わせフォームでは内容ごとに`input`と`textarea`を分けた
　→記憶が曖昧だったため調べた。`textarea`を使用することで、複数行の文章に対応できることを知った。
　　→`label`を追加。`id`で`input`との連携も忘れずに。
- `footer`の「フッター1～3」を見出しにした
　→見出しにするか文章にするかで悩み、見出しを選択。また、フッターの見出しは各セクションの見出しと同レベルという認識。
　　→`h3`に変更。フッターの見出しではなくコンテンツタイトルの場合はmain内の見出しと同レベルにしない。「目次にした時に同じレベルに並んでも違和感がないか？」を基準に選定する。
- ページ下部からページトップへ飛ぶボタン
　→`index.html`をリンク先として指定。
　　→`#`に変更。実務ではこの指定が一般的だと知った。
